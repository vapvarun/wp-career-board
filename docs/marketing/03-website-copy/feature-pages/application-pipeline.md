## SECTION 1: THE PROBLEM WITH STATUSES

You have twelve candidates for one role. Three are in screening, four went to interviews, two got offers. In the free version, they're all just "Reviewing" or "Shortlisted" — you can't see the distinction at a glance.

When you're hiring for multiple roles at once, you need columns, not just labels.

---

## SECTION 2: HOW THE PIPELINE WORKS

**Heading:** Your hiring stages, your way

The Pipeline starts with sensible defaults:

**Submitted → Screening → Interview → Offer → Hired / Rejected**

You can rename, reorder, add, or remove any stage. Name them whatever your process uses — Phone Screen, Technical Test, Panel Interview, References, whatever.

Each stage has:
- A color you choose (green for positive stages, red for rejections, neutral for the rest)
- Optional "terminal" flag — marks the stage as a final outcome (Hired or Rejected)

**When a candidate reaches a terminal stage:**
- If Hired — the job's hired counter increments
- If Rejected — the candidate receives an automatic rejection email (if you've enabled it)

---

## SECTION 3: THE KANBAN BOARD

**Heading:** See your entire pipeline at once

Open Employer Dashboard → Applications → Kanban View.

Each column is a stage. Candidate cards show the applicant name, which job they applied to, and how long they've been in the current stage.

Drag a card to move the candidate. Click it to open the full application.

**Bulk moves:** In list view, select multiple candidates and move them all to a stage at once — useful for clearing a screening backlog.

---

## SECTION 4: PER-BOARD PIPELINES

**Heading:** Each board has its own stages

If you're using the Multi-Board Engine, each board configures its pipeline independently. A "Tech Jobs" board might have a coding challenge stage; a "Marketing Jobs" board doesn't.
