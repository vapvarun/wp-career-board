## SECTION 3: STACK (for developer audiences)

**Heading:** Built the right way

**Subheading:** No jQuery. No shortcodes. No `admin-ajax.php`.

Every piece of WP Career Board uses WordPress core APIs the way they were designed to be used.

**Four columns:**

**WordPress Interactivity API**
The frontend runs on the native WP Interactivity API — the same technology that powers the Query Loop block. No React, no Vue, no jQuery. One script, loaded once, handles every dynamic interaction.

**Gutenberg Blocks**
Every page is a block. Job listings, the application form, employer and candidate dashboards, company profiles — all registered via `block.json` and inserted with the block editor. No PHP templates to override.

**REST API**
Every user action — posting a job, submitting an application, updating status — hits a typed REST endpoint at `/wp-json/wcb/v1/`. Every route has `permission_callback`, `validate_callback`, and `sanitize_callback`. Returns `WP_Error`, never throws exceptions.

**WordPress Abilities API**
Permissions are declared with `wp_register_ability()` and checked with `wp_is_authorized()`. No `current_user_can()` calls with capability strings. Auditable, overridable, clean.

## SECTION 7: FAQ

**Q: Is the free version actually free, or does it unlock after a trial?**
A: Free, permanently. The free plugin is a complete job board — not a time-limited trial and not a locked demo. Pro adds advanced features for sites that need them.

**Q: Does it work with any WordPress theme?**
A: Yes. The blocks are theme-agnostic — they inherit your theme's typography and colors. Reign Theme and BuddyX Pro have deeper first-class integration (automatic color and layout matching), but every Gutenberg-compatible theme works.

**Q: Does it replace WP Job Manager?**
A: WP Career Board and WP Job Manager solve the same problem, but differently. WP Career Board is entirely block-based and uses the Interactivity API — there are no shortcodes. If you're starting fresh, WP Career Board is the more future-proof choice. Migrating from WP Job Manager is possible but requires re-inserting blocks.

**Q: Can candidates apply without creating an account?**
A: Yes. Guest applications are enabled by default — candidates submit their name, email, and cover letter without registering.

**Q: Does it slow down my site?**
A: No JavaScript framework is loaded. The frontend runs on the WordPress Interactivity API, which is already part of the WP core bundle. The admin interface uses no additional JavaScript libraries either.

**Q: Can I monetize the job board (charge employers)?**
A: With Pro. The credit system lets you create credit packages at any price, sell them through your WooCommerce store using any payment gateway, and require employers to spend credits to post jobs.
