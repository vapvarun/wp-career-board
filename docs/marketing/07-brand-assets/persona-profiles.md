# Target Personas

---

## Persona 1: The Community Site Owner

**Name:** Priya
**Role:** Owner / admin of a BuddyPress or BuddyBoss community site
**Theme:** Reign Theme (Wbcom Designs)
**Technical level:** Comfortable with WordPress admin, not a developer

### Goals
- Add a job board to her community so members can post and find jobs
- Keep everything in one place — no redirecting users to external job sites
- Have it look like part of the site, not a plugin

### Pain Points
- Existing job board plugins use shortcodes — she doesn't understand where to put them
- Most options require a separate subscription for things that should be free (guest applications, GDPR, etc.)
- Setup takes too long and requires reading a 40-page manual

### Why WP Career Board Appeals
- She already has Reign Theme — WP Career Board is bundled in her license
- Setup wizard creates all the pages in one click
- BuddyPress integration means job posts show up in the community activity feed automatically

### Key Message
> "It came with Reign Theme, I ran the Setup Wizard, and my job board was live in ten minutes."

## Persona 2: The WordPress Agency Developer

**Name:** Marcus
**Role:** Lead developer at a 5-person WordPress agency
**Technical level:** Advanced — writes PHP and JavaScript, familiar with WP internals

### Goals
- Build a production-quality job portal for a client without starting from scratch
- Use modern WP patterns (blocks, REST API, Interactivity API) — not legacy shortcodes and jQuery
- Something he can hand to the client's admin with documentation they can actually follow

### Pain Points
- Most job board plugins are built on jQuery and shortcodes — they fight the block editor
- Plugins that use React add an entire framework to the page when WP already has one (Interactivity API)
- Client admins get confused by complex settings screens

### Why WP Career Board Appeals
- Entirely block-based — works with any Gutenberg-compatible theme
- REST API and Abilities API means he can extend it cleanly without hacking core files
- The wbcom-admin-wrapper settings UI is clean enough to hand to a non-technical client

### Key Message
> "Built on the Interactivity API and Abilities API — the architecture you'd design yourself if you had time to build from scratch."

## Persona 3: The Niche Job Board Owner

**Name:** David
**Role:** Solo founder running a niche job board (e.g., remote design jobs, nonprofit careers)
**Technical level:** Intermediate — manages WordPress, uses Elementor or block themes

### Goals
- Monetize the job board by charging employers to post listings
- Keep operating costs low — avoid per-seat SaaS pricing
- Attract candidates with a clean, fast job search experience

### Pain Points
- SaaS job board platforms take a cut or charge per posting
- WordPress job board plugins with paid listings often require multiple add-ons
- Site speed suffers when adding feature-heavy plugins

### Why WP Career Board Appeals
- Pro Credit System with Stripe lets him charge employers for job posts — no per-transaction platform fee
- Interactivity API frontend is fast — good Core Web Vitals without extra optimization work
- Multi-Board Engine (Pro) lets him run separate boards (e.g., "Full-Time", "Contract", "Remote") from one install

### Key Message
> "Charge employers directly via Stripe, no marketplace middleman. Run as many boards as you need from one WordPress install."

## Persona 4: The Startup Founder / Internal HR

**Name:** Leila
**Role:** Founder or Operations lead at a 20-50 person startup
**Technical level:** Low — uses WordPress for the company site, not technical

### Goals
- Post open positions on the company website without managing a separate ATS
- Let hiring managers review applications in one place
- Automate the "we received your application" email

### Pain Points
- Paid ATS tools are expensive and overkill for a small team
- Copying job posts to multiple platforms (LinkedIn, company site) is time-consuming
- Candidates never know the status of their application

### Why WP Career Board Appeals
- Pro Application Pipeline gives her a Kanban board to track candidates from Submitted → Hired
- Automatic status-change emails keep candidates informed without manual follow-up
- Setup Wizard has the whole thing running in minutes

### Key Message
> "A hiring pipeline on your own website — applications tracked in a Kanban board, status emails sent automatically."
